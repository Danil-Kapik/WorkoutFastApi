🎯 СООТВЕТСТВИЕ BACKEND
═════════════════════════════════════════════════════════════════════════════

Backend: WorkoutSessionReadSchema     Frontend: SessionResponse
───────────────────────────────────────────────────────────────────────────
✅ id                                  → key={session.id}
✅ user_id                             → (передаётся через пропсы)
✅ exercise_type                       → <h3>{session.exercise_type}</h3>
✅ difficulty                          → Сложность: {difficulty}
✅ reps_per_set_at_start              → Повторения: {reps_per_set_at_start}
✅ completed                           → Status badge / className
✅ notes                               → {session.notes && <notes>}
✅ created_at                          → Дата: {formatDate(created_at)}
✅ updated_at                          → (не отображается, но в типе)

Backend: PaginatedResponse             Frontend: SessionListResponse
───────────────────────────────────────────────────────────────────────
✅ items[]                             → {data.items.map(...)}
✅ total                               → Всего: {data.total}
✅ page                                → Страница {data.page} из...
✅ size                                → (используется в API)
✅ pages                               → ...из {data.pages}
✅ has_next                            → disabled={!has_next}
✅ has_prev                            → disabled={!has_prev}
